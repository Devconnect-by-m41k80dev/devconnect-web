DevConnect
Documentación Web

Un frontend Next.js de nivel producción para conectar desarrolladores y colaborar en proyectos de código abierto del mundo real. Construido con App Router, TypeScript, Tailwind CSS y Zustand.

Next.js 16
TypeScript 5
React 19
Tailwind 3
Zustand 5
Axios
Zod
✦

Características

🔐
Autenticación por Modal
Inicio de sesión y registro en un modal superpuesto — sin páginas de auth dedicadas. Funciona desde cualquier ruta. Sesión con cookie HttpOnly y refresco silencioso en 401.
🛡️
Middleware de Edge
Protección de rutas mediante proxy.ts ejecutado en el Edge de Next.js. Decodifica el JWT, verifica el rol y redirige a usuarios no autenticados antes de renderizar la página.
🔔
Doble Sistema de Notificaciones
Insignia roja para dueños de proyectos con solicitudes pendientes. Insignia verde para postulantes aceptados. El estado de aceptación se persiste en localStorage — se limpia cuando el usuario visita su perfil.
🎯
Descubrimiento Inteligente
Feed de proyectos con coincidencia por habilidades que muestra puntuaciones del algoritmo de relevancia del backend. Los proyectos se ordenan según el stack tecnológico y rol profesional del usuario.
💬
Integración con Discord
Los miembros aceptados ven un CTA de Discord en la página de detalle del proyecto y en su lista de solicitudes del perfil, redirigiéndolos al espacio de colaboración comunitaria.
🌐
Internacionalización
Soporte completo en inglés / español mediante un store i18n personalizado basado en Zustand. Todas las páginas, formularios, páginas legales y el pie de página están completamente traducidos. El idioma se persiste en localStorage.
🎨
Sistema de Temas
Modo claro / oscuro con transiciones suaves de variables CSS y script de prevención de parpadeo. Los colores se definen como propiedades personalizadas CSS — nunca como valores hex hardcodeados.
📱
Totalmente Responsivo
Diseño mobile-first con un menú de navegación móvil personalizado a pantalla completa. Cada diseño se prueba a 375px, 768px y 1280px de ancho.

La regla de oro

Las páginas son delgadas. Los hooks manejan datos. Los componentes manejan el renderizado.
Un page.tsx llama a un hook y compone algunos componentes. Nunca contiene useState para datos, useEffect con llamadas fetch, ni importaciones directas de axios. Todo eso pertenece a hooks/.

Capa HTTP

Tres niveles de abstracción aíslan responsabilidades y hacen que la capa API sea completamente testeable y reemplazable.

1
lib/api/*.api.ts — Módulos por recurso
Un archivo por recurso del backend: projects.api.ts, auth.api.ts, users.api.ts, follows.api.ts, stats.api.ts, metadata.api.ts. Todos exportados desde lib/api/index.ts.
2
lib/http/http-methods.ts — Wrappers tipados
Wrappers tipados get<T>, post<T>, patch<T>, del<T>. Lanzan un Error con el mensaje del backend si success === false.
3
lib/http/http-client.ts — Instancia de Axios
Instancia de Axios con baseURL desde NEXT_PUBLIC_API_URL y withCredentials: true. El flag withCredentials es esencial — sin él, el navegador no enviará la cookie HttpOnly de autenticación.
4
lib/http/interceptors.ts — Procesamiento de respuestas
Desenvuelve el sobre { success, data, message } del backend. En caso de 401: llama silenciosamente a POST /auth/refresh y reintenta la solicitud original. Si el refresco falla: cierra sesión y despacha el CustomEvent open-auth-modal.
🔐

Flujo de Autenticación

1
El usuario hace clic en "Iniciar Sesión"
useModal().openAuth("login") → ModalContext establece authOpen = true → <AuthModal> se renderiza con el formulario de login.
2
POST /auth/login
Se envían las credenciales. El backend establece cookies HttpOnly token y refreshToken. El frontend llama a GET /users/me, guarda el usuario en useAuthStore. El modal se cierra.
3
Navegar a una ruta protegida
proxy.ts (middleware Edge) decodifica la cookie JWT y extrae el role. Si está autenticado → se renderiza la página. Si no → redirige a /?auth=login&redirect=/ruta-original. ModalContext lee ?auth al montarse y abre el modal de login.
4
Refresco silencioso de 401
Cualquier solicitud devuelve 401 → interceptors.ts llama a POST /auth/refresh → reintenta la solicitud original. Si el refresco falla → logout() + CustomEvent open-auth-modal → el modal de login se abre sin ninguna navegación de página.
🔔

Sistema de Notificaciones

Dos flujos de notificaciones independientes comparten un store de Zustand (notifications.store.ts).

🔴 Notificaciones del dueño — solicitudes pendientes
1
GET /projects?limit=100 → filtrar por owner.id === userId
2
Promise.allSettled → GET /projects/:id/applications en paralelo para cada proyecto propio
3
Contar status === "pending" → establecer pendingCount → 🔴 insignia roja en el avatar. TTL de caché: 60s. decrement() se llama tras cada aceptar/rechazar del dueño.
🟢 Notificaciones del postulante — solicitudes aceptadas
1
GET /projects/applied → filtrar status === "accepted"
2
Filtrar applicationId NOT IN localStorage("dc-accepted-seen") → establecer acceptedCount → 🟢 insignia verde
3
El usuario abre /profile → markAcceptedSeen() → guarda IDs aceptados en localStorage → acceptedCount = 0 → la insignia desaparece
🗄

Gestión de Estado

El estado global usa Zustand. El estado de funcionalidades usa hooks personalizados. Un React Context para el modal de autenticación.

auth.store.ts
Persistido
dc-auth (solo usuario)
Identidad del usuario, login, logout, fetchMe. isAuthenticated nunca se persiste — se deriva de user !== null al rehidratar.
notifications.store.ts
No persistido
Se recalcula al montar
pendingCount (dueño) + acceptedCount (postulante). Caché con TTL de 60s. decrement() para actualizaciones optimistas.
theme.store.ts
Persistido
dc-theme
Alternancia claro / oscuro. Agrega/elimina la clase .dark en <html>. Script inline de prevención de parpadeo en layout.tsx.
useLocaleStore (i18n)
Persistido
dc-locale
Preferencia de idioma EN / ES. Usado por useI18n() para devolver el diccionario correcto.

Internacionalización

Soporte completo en inglés / español mediante un sistema personalizado basado en Zustand en src/app/i18n/index.ts. El objeto t está completamente tipado — TypeScript lanza errores si se referencia una clave inexistente.

tsx // En cualquier componente:
const { t, locale, setLocale } = useI18n()

// Acceder a una cadena traducida:
<p>{t.profile.memberSince}</p>

// Cambiar idioma:
setLocale('es')

Contribuir

1
Fork y Clonar
Hacé un fork del repositorio en GitHub y cloná tu fork localmente. Agregá el remote upstream: git remote add upstream https://github.com/M41k80/devconnect-web.git
2
Crear una Rama
git checkout -b feat/nombre-de-tu-funcionalidad — creá la rama desde un main actualizado.
3
Escribir Código
Seguí la arquitectura: páginas delgadas, hooks para datos, componentes para renderizado. Sin tipos any. Variables CSS para colores. Cadenas tanto en EN como en ES en el i18n.
4
Commit (Conventional Commits)
feat(notifications): add green badge for accepted apps
fix(auth): prevent double modal on 401
style(landing): improve dark mode contrast
5
Abrir un Pull Request
Subí tu rama y abrí un PR contra main. Asegurate de que yarn lint y yarn build pasen. Probá en modo claro + oscuro y en móvil.
