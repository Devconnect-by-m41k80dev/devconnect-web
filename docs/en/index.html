A production-grade Next.js frontend for connecting developers to collaborate on real-world open source projects. Built with App Router, TypeScript, Tailwind CSS, and Zustand.

TypeScript 5

Features

🔐
Modal-Based Auth
Login and registration in a modal overlay — no dedicated auth pages. Works from any route. HttpOnly cookie session with silent refresh on 401.
🛡️
Edge Middleware
Route protection via proxy.ts running on the Next.js Edge. Decodes JWT, checks role, and redirects unauthenticated users before the page renders.
🔔
Dual Notifications
Red badge for project owners with pending applications. Green badge for applicants who were accepted. Accepted state persisted to localStorage — clears when user visits profile.
🎯
Smart Discovery
Skill-matched project feed that displays scores from the backend's relevance algorithm. Projects ranked by match with the user's tech stack and professional role.
💬
Discord Integration
Accepted members see a Discord CTA on the project detail page and in their profile applications list, routing them to the community collaboration space.
🌐
Internationalization
Full English / Spanish support via a custom Zustand-based i18n store. All pages, forms, legal pages, and the footer are fully translated. Locale persisted to localStorage.
🎨
Theme System
Light / dark mode with smooth CSS variable transitions and inline flash-prevention script. Colors defined as CSS custom properties — never hardcoded hex values.
📱
Fully Responsive
Mobile-first design with a custom full-screen mobile navigation menu. Every layout tested at 375px, 768px, and 1280px viewports.

Getting Started

Prerequisites

You need the following installed on your system before getting started:

Node.js ≥ 18
Yarn ≥ 1.22
DevConnect API running
Git

Installation

bash # 1. Clone the repository
git clone https://github.com/M41k80/devconnect-web.git
cd devconnect-web

# 2. Install dependencies
yarn install

Configure Environment

bash cp .env.local.example .env.local
# Edit .env.local — set NEXT_PUBLIC_API_URL

Run the App

bash # Development (hot reload)
yarn dev

# Production
yarn build && yarn start

🌐 App available at http://localhost:3001
⚠️ The DevConnect API must be running at http://localhost:3000/api first.

⚙

Environment Variables

Copy .env.local.example to .env.local and fill in your values. Only one variable is required.

NEXT_PUBLIC_API_URL
Base URL of the DevConnect API — no trailing slash. Embedded in the client bundle at build time.
http://localhost:3000/api

⚠️ NEXT_PUBLIC_* variables are baked into the build at build time, not runtime. They must be set before running yarn build. Never put secrets in NEXT_PUBLIC_* variables — they are visible in the browser.

The golden rule

Pages are thin. Hooks own data. Components own rendering.
A page.tsx calls one hook and composes a few components. It never contains useState for data, useEffect with fetch calls, or direct axios imports. All that belongs in hooks/.
🔌

HTTP Layer

Three levels of abstraction isolate concerns and make the API layer fully testable and replaceable.

1
lib/api/*.api.ts — Resource modules
One file per backend resource: projects.api.ts, auth.api.ts, users.api.ts, follows.api.ts, stats.api.ts, metadata.api.ts. All exported from lib/api/index.ts.
2
lib/http/http-methods.ts — Typed wrappers
Typed get<T>, post<T>, patch<T>, del<T> wrappers. Throw an Error with the backend's message string if success === false.
3
lib/http/http-client.ts — Axios instance
Axios instance with baseURL from NEXT_PUBLIC_API_URL and withCredentials: true. The withCredentials flag is essential — without it, the browser won't send the HttpOnly auth cookie.
4
lib/http/interceptors.ts — Response processing
Unwraps the backend's { success, data, message } envelope. On 401: silently calls POST /auth/refresh, retries the original request. If refresh fails: logs out and dispatches open-auth-modal CustomEvent.
🔐

Authentication Flow

1
User clicks "Sign In"
useModal().openAuth("login") → ModalContext sets authOpen = true → <AuthModal> renders with the login form.
2
POST /auth/login
Credentials sent. Backend sets HttpOnly token and refreshToken cookies. Frontend calls GET /users/me, sets user in useAuthStore. Modal closes.
3
Navigate to protected route
proxy.ts (Edge middleware) decodes the JWT cookie, extracts role. If authenticated → page renders. If not → redirect to /?auth=login&redirect=/original-path. ModalContext reads ?auth on mount and opens the login modal.
4
Silent 401 refresh
Any request returns 401 → interceptors.ts calls POST /auth/refresh → replays original request. If refresh fails → logout() + open-auth-modal CustomEvent → login modal opens without any page navigation.
🔔

Notification System

Two independent notification streams share one Zustand store (notifications.store.ts).

🔴 Owner notifications — pending applications
1
GET /projects?limit=100 → filter by owner.id === userId
2
Promise.allSettled → parallel GET /projects/:id/applications for each owned project
3
Count status === "pending" → set pendingCount → 🔴 red badge on avatar. Cache TTL: 60s. decrement() called after each owner accept/reject.
🟢 Applicant notifications — accepted applications
1
GET /projects/applied → filter status === "accepted"
2
Filter applicationId NOT IN localStorage("dc-accepted-seen") → set acceptedCount → 🟢 green badge
3
User opens /profile → markAcceptedSeen() → saves accepted IDs to localStorage → acceptedCount = 0 → badge disappears
🗄

State Management

Global state uses Zustand. Feature state uses custom hooks. One React Context for the auth modal.

auth.store.ts
Persisted
dc-auth (user only)
User identity, login, logout, fetchMe. isAuthenticated is never persisted — derived from user !== null on rehydration.
notifications.store.ts
Not persisted
Recalculated on mount
pendingCount (owner) + acceptedCount (applicant). 60s TTL cache. decrement() for optimistic updates.
theme.store.ts
Persisted
dc-theme
light / dark toggle. Adds/removes .dark class on <html>. Flash-prevention inline script in layout.tsx.
useLocaleStore (i18n)
Persisted
dc-locale
EN / ES language preference. Used by useI18n() to return the correct dictionary.

Internationalization

Full English / Spanish support via a custom Zustand-based system in src/app/i18n/index.ts. The t object is fully typed — TypeScript errors if you reference a missing key.

tsx // In any component:
const { t, locale, setLocale } = useI18n()

// Access a translated string:
<p>{t.profile.memberSince}</p>

// Switch language:
setLocale('es')

Push your branch and open a PR against main. Ensure yarn lint and yarn build pass. Test in light + dark mode and on mobile.
